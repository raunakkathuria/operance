# Operance v0.1.0-beta.1

Status: Release notes draft  
Type: Fedora KDE Wayland developer-beta release  
Audience: Early developers, contributors, maintainers

---

## Summary

`v0.1.0-beta.1` is the first Fedora KDE Wayland developer beta for Operance.
It keeps the public support contract narrow: local-first, Linux first, source
checkout first, RPM `mvp` runtime second, and tray plus click-to-talk as the
default interaction model.

This release is not a broad consumer desktop launch. It is intended for
developers who can run documented commands, collect support bundles, and report
reproducible failures.

---

## Supported In This Beta

- Source-checkout bring-up through `./scripts/install_linux_dev.sh --ui --voice`
  and `./scripts/run_mvp.sh`.
- Fedora RPM `mvp` package path with live execution defaults.
- Packaged tray UI and STT runtime checks through the installed MVP runtime
  smoke.
- Tray click-to-talk on Fedora KDE Wayland.
- A conservative release-verified command subset:
  - `open <app name>` and URL-like launch targets such as `open localhost:3000`
  - `focus <app name>`
  - `what time is it`
  - `what is my battery level`
  - `wifi status`
  - `what is the volume`
  - `is audio muted`
- Support bundle collection from both source checkout and installed command
  paths.
- Provider scaffolds for Windows and macOS without claiming native delivery on
  those platforms yet.

---

## Not Claimed Yet

- Broad distro or desktop-environment support.
- Windows or macOS runtime delivery.
- Wake-word-first as the default interaction model.
- Fully bundled wake-word or TTS model assets.
- A zero-setup consumer installer.
- The full implemented Linux command inventory as release-verified.

## Validation

Before tagging this release, run:

```bash
./scripts/run_beta_readiness_gate.sh
./scripts/run_beta_readiness_gate.sh --run-package-gate
./scripts/build_package_artifacts.sh --rpm --bundle-profile mvp
rpm -Kv dist/package-artifacts/rpm/operance-0.1.0-1.noarch.rpm
```

Then run the installed desktop smoke in the active Fedora KDE Wayland session:

```bash
./scripts/run_installed_desktop_smoke.sh
```

Manual release smoke still requires clicking the tray icon, speaking a command
such as `open firefox`, and confirming that the desktop action really happens.

## Issue Reports

Attach a support bundle whenever possible:

```bash
operance --support-bundle
```

For source-checkout runs, use:

```bash
.venv/bin/python -m operance.cli --support-bundle
```

Include Fedora version, KDE Plasma version, session type, install method, exact
command or script, expected behavior, and actual behavior.
