# Operance v0.1.0-alpha.1

Status: Draft release notes  
Type: Public developer-alpha release draft  
Audience: Early adopters, contributors, maintainers

---

## Release Title

`Operance v0.1.0-alpha.1`

---

## Summary

Operance is a local-first desktop action runtime for Linux desktops, with a
shared portable core, per-platform adapters, and an MCP-compatible control
surface.

This release is the first public **developer alpha** for the Linux-first local
core.

Current support contract:

- Fedora KDE Plasma on Wayland
- source checkout is the primary supported path
- RPM install of the `mvp` runtime profile is the secondary supported path
- tray plus click-to-talk is the default interaction model
- wake word is secondary to launch reliability

---

## What Works In This Alpha

Current release-verified command subset on the target Fedora KDE machine:

- `open <app name>`
- URL-like launches such as `open localhost:3000`
- `focus <app name>`
- `what time is it`
- `what is my battery level`
- `wifi status`
- `what is the volume`
- `is audio muted`

Other commands may exist in the broader runtime, but they are not being claimed
as the release-verified alpha surface yet.

## Recommended Quickstart

Use the source-checkout path for the fullest current MVP experience:

```bash
./scripts/install_linux_dev.sh --ui --voice
.venv/bin/python -m operance.cli --version
.venv/bin/python -m operance.cli --doctor
./scripts/run_mvp.sh
./scripts/run_beta_smoke.sh
```

For the Fedora package-backed gate:

```bash
./scripts/run_fedora_alpha_gate.sh
```

---

## What This Release Does Not Claim

This release does not claim:

- support beyond Fedora KDE Wayland
- Windows or macOS delivery
- wake-word-first as the default interaction model
- a fully bundled native package with every optional voice backend and model asset
- a zero-setup consumer install story

The RPM path is valid for the current developer alpha and now vendors the tray
UI plus STT Python runtime needed for click-to-talk. It still requires a human
Fedora KDE desktop smoke for tray interaction, microphone capture, and real app
launch behavior before tagging a packaged alpha release.

## If You Hit Problems

Please attach a support bundle from the same environment where the problem
occurred:

Source checkout:

```bash
.venv/bin/python -m operance.cli --support-bundle
```

Installed package:

```bash
operance --support-bundle
```

Also include:

- Fedora version
- KDE Plasma version
- session type
- whether you used source checkout or RPM
- exact commands or transcripts
- expected versus actual behavior
